# Contributing to Copliot Enigma

We are thrilled that you are interested in contributing to Copliot Enigma! This document outlines the contribution workflow, coding expectations, and support channels for collaborators.

## Code of Conduct

This project and everyone participating in it is governed by the [Copliot Enigma Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to uphold this code. Please report unacceptable behaviour to [maintainers@copliot-enigma.dev](mailto:maintainers@copliot-enigma.dev).

## Ways to contribute

There are many ways to contribute:

- Report issues and regressions
- Propose features or improvements to the Material UI workspace
- Extend Selenium automation scenarios
- Improve developer tooling, tests, or documentation

## Before submitting an issue

- Search [existing issues](https://github.com/danycrafts/copliot-enigma/issues) to avoid duplicates.
- Provide reproduction steps, environment details, and logs whenever possible.
- Use the supplied GitHub issue templates for bug reports and feature proposals.

## Pull request checklist

1. Fork the repository and create a feature branch from `main`.
2. Install dependencies with `pipenv install --dev` and ensure tests pass.
3. Follow our Python style (PEP 8) and keep functions small, typed, and documented.
4. Update or add documentation alongside code changes when relevant.
5. Include screenshots for visual changes when feasible.
6. Run linting and the applicable workflow locally if possible.
7. Submit the pull request with a clear description referencing related issues.

Thank you for helping us build an industrial-grade automation workspace!
